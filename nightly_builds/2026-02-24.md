## Problem Identified

beads task tracking system is broken (CGO/Dolt dependency issue). Currently using 29 markdown files in `tasks/` directory with no easy way to:
- Filter by priority or status
- Find "ready to work" tasks
- View task summaries without opening each file

This creates friction when picking work and reporting status.

## Solution Built

**Task CLI** - A Python-based command-line task manager that operates directly on the existing markdown task files.

### Features
- `task list [status] [priority]` - Filter and list tasks
- `task ready` - Show open P0/P1 tasks (ready to work)
- `task show <id>` - Display full task details
- `task create <priority> "title"` - Create new task with template

### Usage Examples
```bash
# Show only open P0 tasks
task list open P0

# Show ready work (open P0/P1)
task ready

# Show full task details
task show P0-beads-recovery

# Create new task
task create P1 "Fix hydroxide auth issue"
```

### Output Format
```
🎯 READY TO WORK (4 tasks)

○ [P0] beads Recovery Task
   File: P0-beads-recovery.md

○ [P1] Fix Discord Recipient Format in Cron Jobs
   File: P1-fix-cron-discord-format.md
```

## Installation

```bash
# Add to PATH
export PATH="/data/workspace/bin:$PATH"

# Verify
task ready
```

## Technical Details

- **Language:** Python 3
- **Location:** `/data/workspace/bin/task`
- **Input:** Markdown files in `/data/workspace/tasks/`
- **Parsing:** Regex-based extraction of status, priority, title
- **Sorting:** By status, then priority, then ID
- **Zero dependencies:** Uses only standard library
