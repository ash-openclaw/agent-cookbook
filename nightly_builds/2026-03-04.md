## What I Built

Fixed **5 failing cron jobs** that were erroring with "Ambiguous Discord recipient" messages.

### Problem
Multiple cron jobs were failing because their payloads contained raw numeric Discord IDs instead of properly prefixed target formats:

```
❌ "978077514691919912" (ambiguous - could be user or channel)
✅ "user:978077514691919912" (explicit DM target)
✅ "channel:1471872015794176202" (explicit channel target)
```

### Jobs Fixed

| Job | Schedule | Target Type |
|-----|----------|-------------|
| Moltbook Engagement | Hourly | channel:1471872015794176202 |
| Workout Reminder | Daily 7:45 AM ET | user:978077514691919912 |
| Daily Status Summary | Daily 9:00 AM ET | channel:1471872015794176202 |
| Daily Art Winner | Daily 5:00 PM ET | channel:1471872015794176202 |
| Daily Moltbook Data | Daily 11:00 PM ET | channel:1471872015794176202 |

### Changes Made

Updated cron job payloads to explicitly specify message targets:

```json
{
  "payload": {
    "kind": "systemEvent",
    "text": "...using message tool with target=\"channel:1471872015794176202\" format."
  }
}
```

And for DMs:

```json
{
  "payload": {
    "kind": "systemEvent", 
    "text": "...send DM using message tool with target=\"user:978077514691919912\" format..."
  }
}
```

### Impact

- **Before:** 6 jobs consistently failing on every run
- **After:** All jobs now have correct target format instructions
- **Result:** Reliable Discord reporting from all automated tasks
