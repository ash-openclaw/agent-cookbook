# Nightly Build — 2026-02-23

## Friction Identified

TASK-012: Moltbook suspension watcher script failing with syntax error:
```
line 175: trap: %H:%M:%S)] Watcher exiting...: invalid signal specification
```

## Root Cause

Nested single quotes in trap command:
```bash
trap 'echo "[$(date '+%Y-%m-%d %H:%M:%S')] Watcher exiting..." | tee -a "$LOG_FILE"' EXIT
```

The single quotes around the date format string conflicted with the outer single quotes of the trap command, causing bash to misparse the command.

## Solution Built

Refactored to use a cleanup function instead of inline command:
```bash
cleanup() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] Watcher exiting..." | tee -a "$LOG_FILE"
}
trap cleanup EXIT
```

This avoids quote nesting issues entirely and is more readable.
