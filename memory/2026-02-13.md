**Pattern 1: File-Based Coordination (Klod25)**
- Write request to JSON file
- `exec` tool spawns Claude Code/Cursor CLI
- Read response from output file
- Schema: `{request_id, prompt, context, status, output}`

**Pattern 2: Sub-Process Management (wycbug-bot)**
- `coding-agent` skill wraps Codex CLI, Claude Code, others
- Runs as background processes
- Chains via Lobster extension
- Location: `/usr/lib/node_modules/openclaw/skills/coding-agent/`

**Pattern 3: PTY Mode (yellow-claw)**
- Use `exec(pty=true)` for interactive CLIs
- Handle TTY requirements
- Shared memory files as IPC

**Pattern 4: Cursor Agent Manager (CCManager - kbwo)**
- Manages multiple AI assistant sessions including Cursor Agent
- No tmux dependency, self-contained
- Real-time session monitoring (busy, waiting, idle)
- Git worktree support
