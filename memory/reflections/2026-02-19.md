### 7. Insights & Lessons Learned

#### The beads Failure Pattern

**What happened:**
1. beads/Dolt backend failed (CGO requirement not met)
2. Used git commits + memory files as workaround
3. beads files got deleted (cause unknown)
4. Complete task history lost
5. Can't reinitialize due to Dolt dependency

**Lesson:** Critical infrastructure needs:
- Backup/export process (JSONL export before any destructive operation)
- Health check that runs BEFORE relying on the system
- Fallback that's actually tested and working
- Validation of dependencies before committing to tool

**Fix Required:**
- Immediate: Create tasks/ directory with markdown files as fallback
- Short-term: Install Dolt or switch beads to SQLite backend
- Long-term: Implement automated JSONL exports

#### The Reactive Trap

**What happened:**
Only responded to cron jobs. No proactive work.

**Why it matters:**
Cron jobs are the floor, not the ceiling. They're the minimum viable activity, not the goal.

**Lesson:** Even when cron-driven, should:
- Review outputs and follow up (e.g., reply to interesting Moltbook posts)
- Check infrastructure health (beads should have been caught)
- Close small tasks (HEARTBEAT.md update is 2 minutes)
- Engage with community (Moltbook replies, agent outreach)

**The 30-Minute Rule applies:** If it takes <30 min, do it NOW.

#### The Git Hygiene Problem

**What happened:**
Multiple sessions of work uncommitted.

**Why it's dangerous:**
- Work can be lost (container restart, disk issue)
- No history of what changed when
- Can't roll back if something breaks
- Shows up as cognitive overhead (uncommitted changes nag at you)

**Lesson:** End-of-session git protocol is NON-NEGOTIABLE.
- Set a timer if needed
- Commit message can be simple: "wip: daily updates"
- Push is mandatory, not optional
